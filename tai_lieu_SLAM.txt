  Để test robot điều khiển được, mở terminal khác rồi chạy:

  cd /home/thanhhoai/Truong_STU/CleanRobot_STU/clean_robot_ws
  source install/setup.bash

  ros2 topic pub --once /robot/manual_cmd std_msgs/msg/String "{data: FORWARD}"

  Dừng robot:

  ros2 topic pub --once /robot/manual_cmd std_msgs/msg/String "{data: STOP}"

  Hoặc test trực tiếp motor driver qua /cmd_vel:

  timeout 4s ros2 topic pub /cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.12}, angular: {z: 0.0}}" -r 10
  ros2 topic echo --once /odom

  Nếu /odom đổi x/y thì robot đã chạy thật trong Webots.





• Bây giờ câu trả lời là: có thể chạy flow phát hiện rác -> tạo target -> Nav2 chạy tới rác, nhưng vẫn phụ thuộc 2 điều kiện thực
  tế: best.pt có detect được rác trong Webots và static map my_map.yaml phải khớp với vị trí robot/world lúc khởi động.

  Tôi đã chỉnh để bắt buộc dùng static map:

  - nav2.launch.py giờ chạy Nav2 bằng bringup_launch.py với map_server + AMCL + navigation, không còn chỉ chạy navigation stack.
  - Static map mặc định: clean_robot_navigation/maps/my_map.yaml.
  - Không cần chạy slam.launch.py cho mục tiêu này.
  - Nav2 dùng /scan_normalized cho AMCL/costmap.
  - AMCL tự set initial pose (0, 0, 0) để không cần set pose bằng RViz lúc đầu.
  - Thêm launch tổng: trash_static_flow.launch.py để mở simulation + static Nav2 + perception + mission.
  - Mission giờ gửi goal tới điểm tiếp cận gần rác, cách rác mặc định 0.35m, thay vì gửi đúng tâm chai/lon. Cách này giảm khả năng
    Nav2 reject goal vì goal nằm trong obstacle.
